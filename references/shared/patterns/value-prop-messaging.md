# Pattern: Value Prop Messaging

## Applicable Situation

Use when turning capabilities, positioning, or ICP insight into acquisition, sales, or launch messaging.

## Inputs

- ICP and buyer champion
- Pain or trigger
- Differentiated value
- Proof points
- Channel and format

## Default Workflow

1. Start with the buyer's urgent problem or trigger.
2. State the differentiated outcome in buyer language.
3. Tie the outcome to a specific workflow, metric, cost, risk, or revenue lever.
4. Add proof: customer evidence, benchmark, artifact, use case, or credible mechanism.
5. Adapt the message to the channel: outbound, website, sales deck, launch, or content.
6. Keep objections out of the core value proposition unless the channel is explicitly risk-handling.

## Branch Rules

- For outbound, lead with relevance and trigger, not a full pitch.
- For PLG conversion, connect message to activation and time-to-value.
- For enterprise sales, give the champion language they can use internally.

## Anti-Patterns

- Listing features without "so what."
- Using broad claims any competitor can make.
- Sending the same message to every segment.

## Acceptance Criteria

- The message is specific to a buyer, problem, and outcome.
- It can be tested in a channel.
- It includes or points to proof.

## Conflict Notes

Messaging for acquisition should be sharper and shorter than product education. Product detail belongs in follow-up proof, not the first hook.
